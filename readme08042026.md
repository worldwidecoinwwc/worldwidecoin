# WorldWideCoin Blockchain - README (April 8, 2026)

## Overview

WorldWideCoin is a full-stack blockchain project built in Python with proof-of-work mining, UTXO transaction support, P2P networking, REST API access, and both desktop and mobile wallet integrations. This repository demonstrates a complete cryptocurrency system from consensus rules to user-facing applications.

## Project Website

This project is linked to the public GitHub Pages repository at:

- https://github.com/worldwidecoinwwc/worldwidecoinwwc.github.io

Visit the website repo for the project landing page, hosted docs, and additional project resources.

## What’s Included

- Proof-of-work blockchain with dynamic difficulty
- UTXO transaction model with ECDSA signatures
- P2P networking and chain synchronization
- Flask REST API for wallet and node access
- Command-line interface for node control
- SQLite-backed persistent storage
- Web dashboard explorer
- Mobile wallet with encrypted backup and restore
- Docker deployment for multi-node networks

## Core Components

### Blockchain Engine
- `core/block.py` — Block format, hash calculation, proof-of-work
- `core/blockchain.py` — Chain rules, validation, and consensus logic
- `core/transaction.py` — Transaction creation, signing, and verification
- `core/utxo.py` — UTXO tracking and balance management

### Networking
- `network/node.py` — Node lifecycle and peer operations
- `network/p2p.py` — Peer discovery, connection handling, message broadcasting
- `network/protocol.py` — Peer-to-peer network protocol definitions

### Storage
- `storage/chain_store.py` — Block persistence and retrieval
- `storage/mempool.py` — Transaction pool tracking and indexing

### APIs and Interfaces
- `api/rest_api.py` — Flask REST API server for external clients
- `cli/cli.py` — Command-line interface for node management and wallet operations
- `mobile/` — Mobile-first wallet web app with secure key backup/import
- `explorer/` — Browser-based dashboard for blocks, txs, and network status

## API Endpoints

The REST API supports the following core endpoints:

- `GET /health` — API health check
- `GET /status` — Node status summary
- `GET /metrics` — Node and mempool metrics
- `GET /chain` — Retrieve the blockchain
- `GET /block/<hash>` — Get details for a specific block
- `GET /transaction/<txid>` — Retrieve a transaction
- `GET /utxo/<address>` — Get balance and UTXOs for an address
- `GET /history/<address>` — Get transaction history for an address
- `POST /tx` — Broadcast a signed transaction or submit a legacy send request
- `POST /tx/broadcast` — Broadcast a raw signed transaction
- `GET /peers` — List connected peers
- `POST /peer` — Add a peer to the node
- `GET /mempool` — View pending transactions
- `GET /mining/status` — Mining status and stats

## CLI Commands

Available `python cli/cli.py` commands include:

- `api` — Start the REST API server
- `mine` — Start mining blocks
- `balance <address>` — Check balance for an address
- `send <recipient> <amount>` — Send coins from current wallet
- `chain` — Display chain status
- `peers` — List connected peers
- `connect <host> <port>` — Connect to a peer
- `wallet` — Wallet management commands
- `config` — View and update configuration
- `status` — Node status summary
- `help` — Show available commands

## Mobile Wallet

The mobile wallet is now fully integrated in `mobile/` and includes:

- Local private key generation
- Private key import via hex string
- Wallet balance and transaction history
- Transaction building and signing in-browser
- Encrypted backup/export of the private key
- Download encrypted backup files
- Upload encrypted backup files for restore
- Secure restore/import from backup text

### Run the Mobile Wallet

```bash
cd mobile
python -m http.server 8082
```

Then open your browser at:

```text
http://localhost:8082
```

## Security Notes

- Private keys are stored locally in browser `localStorage` for the mobile wallet
- Backup data is encrypted with AES-GCM and a user-provided password
- Restoring verifies the backup address before loading the wallet
- Signed transactions are broadcast only after local signing

## Docker Deployment

Docker support enables easy multi-node setup.

### Quick Start

```bash
cd worldwidecoin
./deploy.sh      # Linux/Mac
# or
deploy.bat       # Windows
```

### Manual Deployment

```bash
docker-compose build
docker-compose up -d node1 api1 node2 api2 node3 api3
```

Optional miner:

```bash
docker-compose --profile miner up -d miner
```

### Access Points

- Node 1 API: http://localhost:8001
- Node 2 API: http://localhost:8002
- Node 3 API: http://localhost:8003
- Web dashboard: http://localhost:8080

## Web Dashboard

A browser-based explorer is available in `explorer/` and can be started with:

```bash
python web_server.py --open
```

If the default dashboard port is occupied, the server will now automatically try the next available port and report that port on startup.

Dashboard features:

- Block explorer
- Transaction viewer
- Wallet interface
- Peer monitor
- Mining status

## Deployment & Usage

### Local Development
```bash
# Setup virtual environment
python -m venv .venv
.venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Run tests
python -m pytest tests/ -v

# Start CLI
python cli/cli.py --help
```

### Network Deployment
```bash
# Start mining node
python cli/cli.py mine

# Start API server
python cli/cli.py api --host 0.0.0.0 --port 5000

# Connect to network
python cli/cli.py connect <peer_host> <peer_port>
```
